Variables de Entorno


Lista de las variables

Para ver la lista de las variables definidas en el shell actual utilizamos el comando -set
-set
Ver el contenido de una variable:
-echo $HOME
Modificar el valor de una variable:
-variable:valor
-echo $variable
-variable = 'palabra.1 palabra.2 '
-echo $variable

HOME
La variable HOME contiene la dirección de el inicio del usuario. No modificar.


PATH
La variable PATH contiene una lista de directorios que el shell explora cuando este debe invocar un comando exteno
-echo $PATH

-find / -name date 2 > /dev/null
-find / -name ping 2 > /dev/null
Busca en el directorio raiz (/) los nombres -date, y otro de nombre -ping mostrandonos donde se encuentran.
Los errores los redirecciona a /dev/null

-alias

Comano -echo
El comando -echo se utiliza para mostrar por pantalla los mensajes que queramos, la opción -e hace que podamos interpretar los carácteres especiales
->\n

echo "Escribimos lo que queramos, salto de línea \ny otro \n el salto de línea. Salto de línea con \ + n"
echo a\nb
Nos imprime directamente -> a\nb , sin salto de línea
Las comillas son obligatorias
echo -e "a\nb"

->\c
Con \ + c eliminamos el salto de línea del comando -echo despues de el último cierre
echo "Primera línea"; echo "Segúnda línea";
echo "Primera línea\c"; echo "Segúnda línea\c";
La opción -e es indispensable para interpretar "\c" correctamente

->\t
Permite mostrar una tabulación

echo "2 tabulaciones \t\tlisto..."
echo -e "2 tabulaciones con la opción -e \t\tlisto..."
Carácteres de escape
\a Tono
\\ Brra invertida
\c Eliminación de salto de línea al final de la línea
\f Salto de página
\b Borrado del carácter anterior
\n Salto de línea
\t tabulación horizontal
\r Retorno de carro

El carácter ~ significa el directorio de el usuario actual

-id
El comando -id muestra el -id de el usuario y el de el grupo al que pertenece

-pwd Con el comando -pwd nos muestra el directorio actual

Copiar ficheros:
Copia el archivo -f1 al directorio actual del usuario
-cp f1 ~

Copia el fichero /tmp/f1 a /usuario/docs
-cp f1 ~/docs
Copia el archivo -f1 en el usuario -sebastian
Si ~ va seguido de un nombre, se considera nombre de usuario
-cp f1 ~sebastian
-ls /home/sebastian/f1
Copia el directorio -f1 en el directorio ~sebatian/dir
-cp f1 ~sebastian/dir

-cd

Cambiar al directorio -sebastian

Volver al directorio anterios con -cd:

Expresiones básicas
-ls
Listar todos los nombres de los archivo que terminen por *.c
-ls *.f
Mostrar todos los nombres de archivo que comiencen por la letra c:
-ls c*
El signo de interrogación -? representa una letra cualquiera
Nos muestra todos los nombres de los archivos que como extensión tengan una sola letra
-ls *.?
Nos muestra todos los nombres de los archivos compuestos por 3 carácteres
-ls ??? Los carácteres [] permiten especificar la lista de carácteres que se esperan en una posición concreta en el nombre del archivo.
Archivos cuyo nombre empiece por -fu u -o y termine por el carácter . seguido de minúscula:
-ls [fo]*.[a-z]
Archivos cuyos nombre tiene en el segundo carácter una mayúscula, una cifra o la letra i. Los dos carácteres tendrán a continuación una cadena cualquiera Nos muestra los nombres de los archivos que empiecen por una letra cualquiera, como segundo carácter sea mayúscula desde la letra A hasta la letra Z,o un número del 0 al 9 o siemplemente sea como segunda letra le letra i minúscula.Seguido de el asterisco (*) que nos permite cualquier cadena de carácteres
-ls ?[A-Z0-9i]*
El signo exclamación (!) nos indica que el contrario de la regla que está entre corchetes en este caso, los archivos que no comiencen por minúsculas de la -a a la -z:
-ls ![a-z]
Eliminar todos los archivos cuyo nombre terminen por .d o .e
-rm -i *.d *.e
?(expresión)
Archivos cuyo nombre comience por "dis" seguido de 0 o 1 ocurrencias de "456", seguido de ".usi"
-ls dis?(456).usi
*(expresión) La expresión estará presente entre 0 y -n veces
-ls dis*(456).usi
+(expresión)
La expresión estará presente entre 1 y -n
veces -ls dis+(456).usi @(expresión)
La expresión estará presente solo una vez
-ls dis@(456).usi
!(expresión)
La expresión no estará presente
-ls dis!(456).usi
Alternativas ?(expresión|expresión|expresión...)
*(expresión|expresión|expresión...)
+(expresión|expresión|expresión...)
@(expresión|expresión|expresión...)
!(expresión|expresión|expresión...)
 Archivos cuyo nombre comience por -dis seguido de -456 o -908 seguido de .usi
-ls dis@(456|908).usi
Archivos cuyo nombre comience por -fic seguio de -1 a -n ocurrencias de -866 o -867 seguido de .usi
-ls dis+(456|908).log

Redirecciones
Las redirecciones nos permiten almacenar el resultado de uno o varios comandos almacenandolos en archivos de texto por ejemplo.
Entrada y salida estándar de los procesos Redirección simple Se envía la salida del comando -pwd al archivo -salida_pwd, si no está el archivo se crea , si existe se sobreescribe
-pwd > salida_pwd
-cat salida_pwd
Redirección doble Añade a la salida de el comando -date a el archivo -resu
-date >> salida_pwd
-cat salida_pwd

Ejecución de un script por un shell hijo: Normalmente en la mayoría de los casos, los scripts hay que ejecutarlos mediante un shell hijo como intermediario.
Se puede ejecutar un script de tres manera diferentes:
Método primero:
Invocamos el comando -ksh diciendole que nos ejecute el script número 1.
-ksh script_primero.sh
Si mirásemos los permisos de ejecución de este scritp nos saldría algo así:
-rw-r--r-- ... script_primero.sh Segundo método:
-ksh < script_primero.sh
Tercer método:
-chmod u+x script_primero.sh
-ls -l script_primero.sh
-rwr--r-- ... script_primero.sh script_primero.sh
Éste método es el más utilizado, con el comando -chmod le aplicamos los permisos necesarios para poder ejecutar lo correctamente

Parámetros posicionales: En los scripts podemos recuperar los parámetros que le pasamos por línea de comandos.
-$#: Número de argumentos recibidos.
-$0: Nombre script.
-$1: Primer argumento.
-$2: Segundo argumento.
-$3: Tercer argumento.
Cuando las variables contienen más de una cifra es obligatorio utilizar los corchetes ${20},${21}, etc,etc
-$* y $@: Representan todos los argumentos
-nano primer_script.sh
echo "Número de argumentos: $#"
echo "Primer argumento: $1"
echo "Segundo argumento: $2"
echo "Tercer argumento: $3"
echo "Todos los argumentos: $*"
Le damos los permisos necesarios para su ejecución:
-chmod u+x primer_script.sh
Llamamos a el script: primer_script.sh pasandole argumentos:
primer_script.sh argumento1 argumento2 argumento3 7654 /dev/null
Ejecucicón de el script:
Número de argumentos: 4
Primer argumento: argumento1
Segundo argumento: argumento2
Tercer argumento: argumento3
7654
/dev/null
